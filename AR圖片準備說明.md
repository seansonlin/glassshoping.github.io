# AR 試戴圖片準備說明

## 問題說明

AR 試戴功能需要使用**只有眼鏡框架的透明背景圖片**（PNG 格式），而不是包含背景、模特兒臉部的完整商品圖片。

## 圖片要求

### ✅ 正確的 AR 試戴圖片
- **格式**: PNG（支援透明背景）
- **內容**: 只有眼鏡框架，無背景、無模特兒
- **角度**: 正面角度，眼鏡水平放置
- **尺寸**: 建議 800x400 或 1000x500（寬高比約 2:1）
- **背景**: 完全透明
- **品質**: 高解析度，邊緣清晰

### ❌ 不適合的圖片
- 包含背景的商品圖片
- 有模特兒臉部的圖片
- JPG 格式（不支援透明背景）
- 側面或傾斜角度的圖片

## 如何準備 AR 試戴圖片

### 方法 1: 使用圖片編輯軟體
1. 使用 Photoshop、GIMP、或線上工具（如 remove.bg）
2. 移除背景，只保留眼鏡框架
3. 確保背景完全透明
4. 調整眼鏡角度為正面水平
5. 匯出為 PNG 格式

### 方法 2: 使用線上工具
- **remove.bg**: https://www.remove.bg/
- **Photopea**: https://www.photopea.com/（免費的線上 Photoshop）
- **Canva**: https://www.canva.com/

### 方法 3: 從供應商取得
- 向眼鏡供應商或製造商索取透明背景的產品圖片
- 通常他們會有專門的產品圖庫

## 圖片範例結構

```
眼鏡框架圖片應該類似這樣：
┌─────────────────┐
│                 │  ← 透明背景
│    👓 框架      │  ← 只有眼鏡框架
│                 │  ← 透明背景
└─────────────────┘
```

## 如何添加到商品數據

在 `src/data/glasses.ts` 中，為每個商品添加 `arImage` 欄位：

```typescript
{
  id: 1,
  name: '經典黑框眼鏡',
  // ... 其他欄位
  image: 'https://...', // 一般商品圖片
  arImage: 'https://...', // AR 試戴專用透明背景圖片（PNG）
  // ... 其他欄位
}
```

## 圖片上傳位置

### 選項 1: 使用 CDN 或圖片託管服務
- 上傳到 Cloudinary、Imgur、或您的 CDN
- 使用圖片 URL

### 選項 2: 使用本地檔案
- 將圖片放在 `public/ar-glasses/` 資料夾
- 使用相對路徑：`/ar-glasses/glasses-1.png`

### 選項 3: 使用圖片 API
- 如果有後端 API，可以透過 API 提供圖片

## 測試透明背景

在瀏覽器中打開圖片 URL，確認：
1. 背景是透明的（不是白色或其他顏色）
2. 只有眼鏡框架可見
3. 圖片清晰，邊緣沒有鋸齒

## 注意事項

1. **CORS 問題**: 如果圖片來自不同網域，需要設定 CORS 標頭
2. **圖片大小**: 建議壓縮圖片以提升載入速度
3. **備用方案**: 如果沒有 `arImage`，系統會自動使用 `image` 作為備用

## 快速測試

可以使用以下透明背景的眼鏡圖片進行測試：
- 搜尋 "glasses png transparent" 在 Google Images
- 使用免費圖庫如 Pixabay、Pexels（搜尋透明背景眼鏡）

## 範例圖片 URL

以下是幾個範例（您需要替換為實際的透明背景眼鏡圖片）：

```typescript
arImage: 'https://example.com/glasses-frame-transparent.png'
// 或
arImage: '/ar-glasses/classic-black-frame.png'
```
